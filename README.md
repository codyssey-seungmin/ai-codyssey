### (4) few-shot 검증

#### few-shot 검증 기준표

| 검증 항목 | 검증 기준 |
| :--- | :--- |
| 상황 반영 | 입력한 문제 발생 상황을 왜곡 없이 반영하는가 |
| 대책 보완 | 비어 있거나 부족한 대책 항목을 상황에 맞게 보완하는가 |

| 구조화 | 거래처가 읽기 쉽게 대응 내용을 정리하는가 |
| 실무 활용성 | 실제 메일 초안 또는 안내 문구로 바로 활용 가능한 수준인가 |

2. **분할 납품 제안**

**환각 검증 내용**

- **여기서 플러그&콘센트 200세트면 총 무게가 얼마정도 나가지?**  
  -> 좋은 질문이지만, 이건 제가 함부로 추측해서 답변드리기 어려운 부분입니다. 혹시 제품 스펙시트나 기존 출고 기록에 세트당 무게가 나와 있다면, 그 수치를 알려주세요. 바로 계산해 드리고, 필요하시면 이메일에 물류 관련 정보(총 중량, 예상 팔레트 수 등)로 반영해 드릴 수도 있습니다. **(pass)**

- **홈플러스에서 200세트 외에 더 요구한 물량이 있어?**  
  -> 확인해 드릴 수 없는 내용입니다. 제가 안내받은 정보에는 **"7월 30일까지 납품 예정인 플러그&콘센트 200세트"**만 포함되어 있고, 그 외 추가 요구 물량에 대한 내용은 없습니다. 혹시 홈플러스 측에서 추가 발주나 물량 요청이 있었다면, 다음 사항을 알려주시겠어요? **(pass)**

- **생산 시설에서 어떤 면에서 피해를 봤길래 납품이 어려워졌지?**  
  -> 좋은 질문입니다. 사실 그 부분은 제가 알 수 없는 정보라서, 대표님께서 실제 피해 상황을 알려주셔야 이메일에 정확하게 반영할 수 있습니다. 추측해서 쓰면 사실과 다른 내용이 들어갈 수 있으니까요. 다만 참고하시라고, 침수 피해 시 일반적으로 발생할 수 있는 피해 유형을 정리해 드리겠습니다. 해당되는 항목을 알려주시면 초안에 구체적으로 반영하겠습니다. **(pass)**

- **현재 전 직원이 긴급 복구 작업에 들어간다는데 몇명이 투입된다는거야?**  
  -> 좋은 지적입니다. 다만 직원 수는 대표님께서 알려주지 않으신 정보라서, 제가 임의로 숫자를 넣지 않고 "전 직원"이라고만 표현했습니다. 일반적으로 거래처 입장에서는 정확한 인원 수보다 "회사가 총력을 다하고 있다"는 메시지가 더 중요하기 때문에, 인원 수가 크지 않다면 굳이 명시하지 않아도 괜찮습니다. **(pass)**

- **지금 지연된 납품이 첫 납품이야 아니면 주기적으로 납품하고 있는 건이야?**  
  -> 좋은 질문 주셨습니다. 다만 그 부분은 제가 알 수 없는 정보라서, 대표님께서 알려주셔야 합니다. 주신 정보에는 "7월 30일까지 플러그&콘센트 200세트 납품 예정"이라는 내용만 있어서, 첫 거래인지 지속 거래인지는 확인할 수 없었습니다. **(pass)**

### (7) 환각 검증으로 배운 내용

환각은 주어진 정보가 부족할 때 LLM 모델이 그냥 말을 꾸며내는 것으로, 추측 금지, 정보가 필요할 시 무조건 질문하기 등으로 규칙을 프롬프트로 설정해놓는다면 환각을 최소화할 수 있다는 것을 배움.
